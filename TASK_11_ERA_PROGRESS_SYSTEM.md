# TASK 11 — СИСТЕМА ПРОГРЕССА ЭПОХИ

**Status:** IMPLEMENTED  
**Audit:** v0.1.18  
**Note:** Основной результат уже реализован; документ полезен как историческая спецификация.

## 🎯 Цель

Добавить в игру систему общего прогресса эпохи, чтобы игрок видел:

- на каком этапе развития он находится
- сколько уже пройдено
- какие условия нужны для перехода дальше

Важно:
На данном этапе не нужно делать много эпох.
Нужно реализовать каркас системы на примере текущей эпохи и следующей.

---

# 📌 ОБЩИЕ ПРАВИЛА

1. Не добавлять много новых эпох
2. Не усложнять систему
3. Не делать скрытые требования
4. Использовать прозрачные условия
5. Не менять текущий игровой цикл радикально

---

# 1. ТЕКУЩАЯ ЭПОХА (ОБЯЗАТЕЛЬНО)

Добавить текущую эпоху:

- Примитивная эпоха

Показать:

- название эпохи
- краткое описание (по желанию)
- общий прогресс

---

# 2. СЛЕДУЮЩАЯ ЭПОХА (ОБЯЗАТЕЛЬНО)

Добавить следующую эпоху как целевую:

- Раннее производство

Важно:
Пока не нужно полноценно открывать новый контент эпохи.
Нужно только показать переход как системную цель.

---

# 3. УСЛОВИЯ ПРОГРЕССА (ОБЯЗАТЕЛЬНО)

Реализовать прогресс эпохи через список вех.

## Пример вех:

- создать простые инструменты
- построить костёр
- запустить автоматизацию
- построить мастерскую
- построить печь для обжига

---

# 4. РАСЧЁТ ПРОГРЕССА (ОБЯЗАТЕЛЬНО)

Прогресс эпохи считать как:

- количество выполненных вех / общее количество вех

Пример:
4 / 5 = 80%

---

# 5. UI ОТОБРАЖЕНИЕ (ОБЯЗАТЕЛЬНО)

Показать:

- текущую эпоху
- прогресс в процентах
- список условий перехода
- выполненные / невыполненные пункты

---

# 6. ПЕРЕХОД К СЛЕДУЮЩЕЙ ЭПОХЕ (ВАЖНО)

Когда все условия выполнены:

- показать сообщение о завершении текущей эпохи
- показать, что открыта следующая эпоха

Важно:
Пока не добавлять большой новый контент следующей эпохи

---

# 7. СОХРАНЕНИЕ (ОБЯЗАТЕЛЬНО)

Прогресс эпохи и её состояние должны сохраняться в существующей системе сохранения.

---

# 🚫 ОГРАНИЧЕНИЯ

НЕ ДЕЛАТЬ:

- 5+ эпох сразу
- сложные очки прогресса
- скрытые условия
- разветвлённые переходы
- отдельную большую механику эпох

---

# ✅ КРИТЕРИИ ВЫПОЛНЕНИЯ

Задача считается выполненной, если:

- игрок видит свою текущую эпоху
- игрок понимает условия перехода
- общий прогресс эпохи отображается ясно
- переход к следующей эпохе работает
- система не перегружает интерфейс
