# TASK 08 — ПОДСКАЗКИ И ПОНЯТНОСТЬ ИНТЕРФЕЙСА

## 🎯 Цель

Сделать интерфейс более понятным для игрока без добавления новых игровых систем.

Игрок должен иметь возможность навести курсор на иконку, кнопку или статус и быстро понять:

- что это такое
- что это даёт
- зачем это нужно

Важно:
Это задача на UX и читаемость, а не на расширение игры.

---

# 📌 ОБЩИЕ ПРАВИЛА

1. Не добавлять новые игровые механики
2. Не перегружать интерфейс лишним текстом
3. Все подсказки должны быть короткими и полезными
4. Подсказки должны работать быстро и стабильно
5. Не использовать большие модальные окна

---

# 1. ПОДСКАЗКИ ДЛЯ РЕСУРСОВ (ОБЯЗАТЕЛЬНО)

## Реализовать:

При наведении на иконку или элемент ресурса показывать tooltip:

### Минимум:

- название ресурса
- краткое описание

### Пример:

Дерево  
Базовый материал для строительства и крафта

Кирпич  
Переработанный материал для построек и развития

---

# 2. ПОДСКАЗКИ ДЛЯ ЗДАНИЙ (ОБЯЗАТЕЛЬНО)

## Реализовать:

При наведении на здание показывать:

- название
- краткое назначение
- что открывает или что автоматизирует

### Пример:

Костёр  
Автоматически превращает глину в кирпич

---

# 3. ПОДСКАЗКИ ДЛЯ ИССЛЕДОВАНИЙ (ОБЯЗАТЕЛЬНО)

## Реализовать:

При наведении на исследование показывать:

- название
- эффект
- зачем оно нужно игроку

### Пример:

Базовые инструменты  
Увеличивают эффективность ручного сбора ресурсов

---

# 4. ПОДСКАЗКИ ДЛЯ СТАТУСОВ И ИНДИКАТОРОВ (ВАЖНО)

## Реализовать:

Подсказки для:

- энергии
- склада
- автоматизации
- очереди крафта

### Цель:

Игрок должен понимать, что означают:

- низкая энергия
- заполненный склад
- состояние “ожидание”
- состояние “работает”

---

# 5. ТРЕБОВАНИЯ К TOOLTIP

## Tooltip должен:

- появляться при hover
- не перекрывать критически важный UI
- быть компактным
- иметь единый стиль
- поддерживать русский текст

## Tooltip не должен:

- занимать слишком много места
- содержать длинные абзацы
- быть отдельной сложной системой

---

# 6. ГДЕ ИСПОЛЬЗОВАТЬ TOOLTIP В ПЕРВУЮ ОЧЕРЕДЬ

Обязательные зоны:

- ресурсы
- здания
- исследования
- очередь крафта
- автоматизация
- энергия / склад

---

# 7. ОГРАНИЧЕНИЯ

НЕ ДЕЛАТЬ:

- энциклопедию
- отдельный справочник
- большие окна помощи
- длинные описания по 5–6 строк

---

# ✅ КРИТЕРИИ ВЫПОЛНЕНИЯ

Задача считается выполненной, если:

- игрок может навести курсор и понять назначение ресурса
- игрок понимает роль здания и исследования
- ключевые статусы UI объяснены
- интерфейс стал понятнее без перегруза
