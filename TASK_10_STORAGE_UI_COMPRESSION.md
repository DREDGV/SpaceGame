# TASK 10 — STORAGE UI COMPRESSION

## 🎯 Цель

Сохранить новую структуру склада с категориями, но сделать панель компактнее, чище и визуально сбалансированнее.

Важно:
Не менять механику склада.
Не убирать категории.
Не менять архитектурную основу.
Нужно исправить именно UI и иерархию информации.

---

# 1. СОКРАТИТЬ ОБЪЁМ ИНФОРМАЦИИ НА ОСНОВНОЙ ПАНЕЛИ

## Оставить на основном экране только:

Для панели склада:

- общий объём: занято / максимум
- общую полосу заполнения
- короткое предупреждение при переполнении

Для каждого ресурса:

- иконку
- название
- количество
- компактную мини-полосу
- индикатор переполнения (если есть)

---

# 2. УБРАТЬ ИЗ ОСНОВНОЙ ПАНЕЛИ И ПЕРЕНЕСТИ В TOOLTIP

Перенести во всплывающую подсказку:

- подробное описание ресурса
- storageSize
- долю от общего склада в процентах
- строку "занимает X места"

---

# 3. СДЕЛАТЬ КАТЕГОРИИ КОМПАКТНЕЕ

## Требования:

- категория = компактная секция
- без длинных описаний в основном виде
- максимум: заголовок + суммарное место по категории

---

# 4. ИСПРАВИТЬ ВЕРТИКАЛЬНЫЙ БАЛАНС

## Требования:

- панель склада не должна чрезмерно растягивать страницу
- уменьшить внутренние отступы
- уменьшить высоту карточек ресурсов
- при необходимости включить внутренний scroll у панели склада

---

# 5. СОХРАНИТЬ ВАЖНЫЕ СИГНАЛЫ

Оставить:

- цветовую индикацию заполнения
- предупреждение о переполнении
- подсветку проблемного ресурса

---

# 6. НЕ ДЕЛАТЬ

- не убирать категории
- не убирать группировку
- не возвращаться к старому плоскому списку
- не добавлять новые механики
- не добавлять новые панели

---

# ✅ КРИТЕРИИ ВЫПОЛНЕНИЯ

Задача считается выполненной, если:

- склад остаётся информативным
- панель стала заметно компактнее
- исчезло ощущение перегруженности
- страница стала визуально ровнее
- подробности доступны через tooltip, а не через постоянный текст
