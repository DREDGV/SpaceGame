# TASK 07 — СИСТЕМА СОХРАНЕНИЯ ПРОГРЕССА

## 🎯 Цель

Добавить в игру локальное сохранение прогресса игрока, чтобы после перезагрузки страницы игра продолжалась с того же состояния.

Важно:
Система должна быть простой, надёжной и не расширять игру новыми механиками.

---

# 📌 ОБЩИЕ ПРАВИЛА

1. Использовать localStorage
2. Не добавлять серверное хранение
3. Не добавлять аккаунты и авторизацию
4. Не усложнять архитектуру
5. Сохранять только актуальные данные основной игры

---

# 1. ЧТО НУЖНО СОХРАНЯТЬ (ОБЯЗАТЕЛЬНО)

## Сохранить состояние:

- ресурсы
- здания
- исследования
- открытые рецепты
- лимит склада
- скидку на крафт
- энергию игрока
- состояние онбординга
- прогресс целей
- очередь крафта
- состояние автоматизации
- журнал можно не сохранять полностью, либо сохранять ограниченно

---

# 2. КОГДА СОХРАНЯТЬ (ОБЯЗАТЕЛЬНО)

## Реализовать автосохранение:

Сохранять после:

- сбора ресурсов
- крафта
- постройки здания
- исследования
- изменения очереди крафта
- завершения онбординга / пропуска
- продвижения по целям

## Дополнительно:

- периодическое автосохранение раз в несколько секунд

---

# 3. ЗАГРУЗКА СОХРАНЕНИЯ (ОБЯЗАТЕЛЬНО)

## При старте игры:

- проверить наличие сохранения
- если сохранение найдено — загрузить его
- если нет — начать новую игру

---

# 4. СБРОС ПРОГРЕССА (ОБЯЗАТЕЛЬНО)

## Добавить:

Простую кнопку или действие:

- "Новая игра"
  или
- "Сбросить прогресс"

## Требования:

- очищает localStorage
- перезапускает игру в начальное состояние
- желательно с подтверждением

---

# 5. ЗАЩИТА ОТ ОШИБОК (ОБЯЗАТЕЛЬНО)

## Реализовать:

- try/catch при сохранении и загрузке
- если сохранение повреждено — игра не должна ломаться
- в случае ошибки загрузки запускать новую игру

---

# 6. UI / UX

## Добавить:

- небольшой индикатор или сообщение:
  - "Прогресс сохранён"
  - или малозаметное автосохранение без спама

## Не нужно:

- больших модальных окон
- сложного меню сохранений

---

# 7. ОГРАНИЧЕНИЯ

НЕ ДЕЛАТЬ:

- серверные сохранения
- несколько профилей
- экспорт/импорт
- облачную синхронизацию
- сложную систему версий

---

# ✅ КРИТЕРИИ ВЫПОЛНЕНИЯ

Задача считается выполненной, если:

- после обновления страницы прогресс сохраняется
- очередь крафта восстанавливается
- здания и исследования сохраняются
- цели и онбординг сохраняются
- игра не ломается при битом сохранении
- есть способ начать новую игру
